## 怎么进入搬瓦工 IP 检测页面

搬瓦工把 IP 检测功能藏在 KiwiVM 控制面板里，不是直接在用户中心首页能看到的，所以不少新手找不到入口。

**方法一：从浏览器地址栏直接跳转（推荐）**

1. 登录搬瓦工账号，进入 Services → My Services，找到你要检测的那台 VPS
2. 点击对应 VPS 后面的 **KiwiVM Control Panel** 按钮，进入控制面板
3. 等页面加载完，看一下浏览器地址栏，URL 格式大概是：`https://kiwivm.64clouds.com/VPSID/main.php`
4. 把地址栏里 `main.php` 这段**替换成** `main-exec.php?mode=blacklistcheck`，回车
5. 页面会出现 **Block List Check** 检测界面，点击 **Test Main IP** 开始检测

## 看懂检测结果

检测完会出现两种结果：

- **IP NOT BLOCKED** → IP 正常，国内可以正常访问
- **IP BLOCKED** → IP 被封，国内无法访问

顺便说一下：官方提示里有一条容易被忽略——中国晚高峰时段（大约晚上 8-11 点）检测结果可能不准，偶尔会误报。建议早上测一次、晚上再测一次，两次都是 BLOCKED 才基本确定封了。

单次结果是 BLOCKED 别立刻换 IP，等等看。

## IP 没被封但还是连不上，怎么回事

这种情况不少见。

IP 检测过了，但 VPS 还是 SSH 登录不了、网站打不开，大概率是**端口被封**而不是 IP 整体被封。GFW 有时候只封掉某个端口，IP 本体还活着。

处理思路：

1. **确认 SSH 端口**：搬瓦工的 SSH 端口不是默认 22，是系统随机分配的，要在 KiwiVM 面板里查。新手容易在这里卡住
2. **用第三方工具测端口通不通**：可以搜索 port.ping.pe 这类工具，输入 IP 和端口号测一下
3. **尝试切换机房**：如果套餐支持迁移，换个机房，IP 和端口都会变

## 怎么降低 IP 被封的概率

这块说几个实际有用的点，不是废话。

IP 被封通常有几个触发原因：流量行为异常被检测、上家租户留下的历史问题、所在机房的 IP 段被批量封。

- **选对机房线路**：CN2 GIA 线路走的是中国电信精品网，GFW 处理起来相对谨慎一些，被封频率比普通 163 骨干线路低
- **不要跑大量对外发包的程序**：扫描工具、爬虫类脚本是 IP 被封的常见原因
- **香港机房注意**：物理距离近，延迟低，但历史上被封频率也偏高，不同时期差异较大

从自己用下来的感受说：CN2 GIA-E 套餐的 DC6 机房整体比较稳，长期下来 IP 被封的次数不算多。当然没有任何机房能完全免疫。

## 常见问题

**Q：搬瓦工 IP 检测工具能检测其他 VPS 的 IP 吗？**

不能。这个工具只针对搬瓦工自己的 IP 地址，输入其他服务商的 IP 是检测不了的。

**Q：IP NOT BLOCKED 但还是连不上，是不是工具有问题？**

工具本身没问题，但它只检测 IP 是否被整体封锁，不检测具体端口。连不上但 IP 显示正常，先检查 SSH 端口是否被封（搬瓦工端口不是 22，要在 KiwiVM 面板查），再考虑端口被单独限制的情况。

**Q：付费换了 IP 之后，原来的 IP 怎么处理？**

原 IP 会回收，你不用管。换完新 IP 之后，记得更新你本地 SSH 配置里的连接地址。

**Q：被封的 IP 能切换机房吗？**

不能。IP 被封状态下，KiwiVM 后台的机房迁移选项会被锁定，必须先换掉封禁的 IP，才能做机房迁移。

**Q：CN2 GIA-E 套餐和 KVM 套餐 IP 被封频率差别大吗？**

有差别，但不是绝对的。CN2 GIA-E 走 CN2 GIA 线路，整体来说被封频率比 KVM 套餐的普通骨干线路低一些，但最终还是取决于你自己的使用方式。
